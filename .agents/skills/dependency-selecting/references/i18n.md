# i18n, Templates & Text Processing

## `github.com/kaptinlin/go-i18n` — Localization

- Token-based and text-based translation
- ICU MessageFormat v1 support
- Flexible loading options
- High performance

**When to use:** Multi-language UI text, localized error messages, translation management.

## `github.com/kaptinlin/messageformat-go` — Message Formatting

- MessageFormat 2.0 (latest Unicode specification)
- ICU MessageFormat v1 implementation
- Full plural rules, select, number/date formatting

**When to use:** Complex pluralization ("1 item" vs "5 items"), gender agreement, locale-aware number/date formatting in translations.

## `github.com/kaptinlin/template` — Template Engine

- Variable interpolation, filters, conditions, loop control
- Liquid/Django-like syntax
- Lightweight

**When to use:** Email templates, notification messages, user-configurable text, CMS content rendering.

## `github.com/kaptinlin/filter` — String & Data Filters

- String manipulation, array/slice operations
- Date/time formatting, number formatting
- Mathematical computations

**When to use:** Template filter pipelines, data formatting, transformation utilities.

## `github.com/agentable/go-humanize` — Human-Readable Formatting

- Byte sizes, relative time, ordinals, comma formatting, percentages, and pluralization
- Pure functions, zero dependencies

**When to use:** Dashboards, CLIs, logs, and other user-facing display of sizes/times/numbers.

## Decision Tree

```
Need text processing?
├── Multi-language support?
│   ├── Simple translations → kaptinlin/go-i18n
│   └── Complex plurals/gender/numbers → kaptinlin/messageformat-go
├── Dynamic content rendering?
│   ├── Full template syntax → kaptinlin/template
│   └── Just data filters → kaptinlin/filter
├── Human-readable display?
│   └── agentable/go-humanize
└── Simple string formatting?
    └── fmt.Sprintf (stdlib)
```

## Do NOT Use

| Library | Reason |
|---------|--------|
| `nicksnyder/go-i18n` | Use `kaptinlin/go-i18n` |
| `text/template` for user content | Security risk. Use `kaptinlin/template` |
