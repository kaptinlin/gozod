# Golang Skills

A collection of Claude Code skills covering Go project workflows — from writing and testing code to linting, committing, releasing, and generating documentation.

| Skill | Description |
|-------|-------------|
| `agent-md-writing` | Generate CLAUDE.md / AGENTS.md for Go projects |
| `code-simplifying` | Refine and simplify Go code for clarity and maintainability |
| `committing` | Create conventional commits for Go packages |
| `config-designing` | Design type-safe configuration with go-config |
| `dependency-selecting` | Select Go dependencies from agentable ecosystem and vetted libraries |
| `go-best-practices` | Google Go coding best practices and style guide |
| `golang-design-guide` | Go design patterns and architectural guidelines |
| `modernizing` | Modernize Go code to use Go 1.20–1.26 features |
| `multimodule-initializing` | Initialize or convert to multi-module Go repos |
| `ralphy-initializing` | Initialize Ralphy AI coding loop config for Go projects |
| `ralphy-tasking` | Create Ralphy TODO.yaml task files for Go projects |
| `readme-writing` | Generate README.md for Go libraries |
| `refactor-global` | Run full-repo high-cohesion refactor governance (global scope) |
| `refactor-modularizing` | Analyze Go codebases and plan structural improvements |
| `refactor-pragmatic-reviewing` | Pragmatic Go architecture review and refactoring |
| `releasing` | Guide the release process with semantic versioning |
| `research-analyzing` | Analyze research and references to generate root ANALYSIS.md |
| `research-collecting` | Find and vendor reference repos as git submodules |
| `research-reporting` | Generate structured research reports from .references projects |
| `research-tasking` | Create research tasks from analysis reports |
| `skill-creating` | Create new Claude Code skills for Go projects |
| `spec-gap-planning` | Plan spec-driven development with gap analysis |
| `spec-gap-tasking` | Create tasks from spec gaps |
| `spec-reviewing` | Audit SPECS directory for consistency and structural issues |
| `spec-tasking` | Create tasks from specifications |
| `spec-writing` | Write specification documents for Go projects |
| `tdd-implementing` | Go TDD: red-green-refactor vertical slices with testify |
| `tdd-planning` | Write TDD-first Go implementation plans from specs or PRDs |
| `tdd-testing` | Write Go tests with testify and Go 1.25+ features |
